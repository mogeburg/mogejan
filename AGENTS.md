# このプロジェクトで絶対に守ること

## 実装前

- 既存実装を調査してから変更する
- store.ts の既存パターンを優先する
- gameplay.ts の既存ルールを優先する
- 推測でゲームルールを変更しない

## ゲームルール

このプロジェクトは一般的な麻雀ではなくドンジャラである。

以下を前提に実装すること。

- このゲームは手牌は8枚＋ツモ1枚。アガリ時に9枚の形になる
- ツモ・ロンには独自条件がある
- 特殊な組み合わせによる独自役が存在する
- 一般的な麻雀ルールへ置き換えてはならない
- リーチをする際に点数は増減しない
- リーチは面前以外の状態（ポンをしている状態）でも使用することが可能
- 面前でリーチした場合は、メンゼン役の追加と裏ドラ表示の権利が追加される

## デバッグ機能

showAllTiles は表示専用。

以下は禁止。

- 勝利判定の回避
- リーチ条件の回避
- ツモ条件の回避
- デバッグフラグによるルール変更

## 状態管理

状態管理の中心は store.ts。

新しいグローバル状態を追加する場合は、

1. store.ts
2. storeSelectors.ts

の既存パターンに従うこと。

## Ability実装

能力関連の処理は gameplay.ts を優先する。

UIコンポーネントへ能力ロジックを書いてはならない。

## 検証

変更後は最低限以下を実行する。

npx tsc --noEmit

ビルドに影響する変更の場合のみ

npm run build

を実行する。
